ServiceNow Virtual Agent

Portable Web Client - テストページ

このテストページについて

このページは、ServiceNow Virtual AgentのPortable Web Clientを外部ウェブサイトに埋め込むためのテンプレートです。

Portable Web Clientを使用すると、ServiceNowインスタンス外の任意のウェブサイトにVirtual Agentチャットウィジェットを統合できます。

セットアップ手順（ServiceNow側の設定）

  1. ServiceNowインスタンスにログイン
  2. All > Virtual Agent > Designer に移動
  3. 使用するVirtual Agentトピックを選択
  4. 「Portable」タブをクリック
  5. 「Website」オプションを選択
  6. 「Generate Code」をクリックして埋め込みコードを生成
  7. 生成されたスクリプトコードをコピー
  8. 下記の「実装方法」に従ってこのHTMLファイルに貼り付け

実装方法

ServiceNowから取得したスクリプトを以下の<!-- SERVICENOW SCRIPT HERE -->の部分に貼り付けてください： 

<!-- SERVICENOW SCRIPT HERE -->
<script>
    // ServiceNowから生成されたスクリプトをここに貼り付け
    // 例:
    const servicenowInstance = 'https://your-instance.service-now.com';

    function loadScript(src, type) {
        return new Promise((resolve, reject) => {
            var script = document.createElement('script');
            script.src = src;
            script.type = type || 'text/javascript';
            script.async = false;
            script.onload = resolve;
            script.onerror = reject;
            document.head.appendChild(script);
        });
    }

    // ServiceNowのチャットスクリプトを読み込み
    loadScript(servicenowInstance + '/uxasset/...');
</script>

必要なServiceNow側の設定

Virtual Agentが正しく動作するために、ServiceNowインスタンスで以下の設定が必要です：

  • CORS (Cross-Origin Resource Sharing) 設定
    • System Web Services > REST > CORS Rules でCORSルールを作成
    • このウェブサイトのドメインを許可リストに追加
  • システムプロパティ設定
    • com.glide.cs.embed.csp_frame_ancestors: このサイトのURLを設定
    • com.glide.cs.embed.xframe_options: sameorigin に設定
  • Portable Web Client の有効化
    • Virtual Agent Designer でPortable Web Clientを有効化

注意: このページを正しく動作させるには、ServiceNowインスタンスから生成された実際の埋め込みコードと、 適切なCORS設定が必要です。セキュリティ上の理由から、ServiceNowインスタンスのURLとトピック設定を正しく行ってください。

カスタマイズオプション

  1. 色のカスタマイズ: チャットウィジェットの色はServiceNow側で設定できます
  2. 位置の調整: ウィジェットの表示位置（右下、左下など）を変更可能
  3. SSO統合: シングルサインオン (SSO) と統合して認証を実装可能
  4. カスタムロゴ: Virtual Agent Designerでカスタムロゴやアバターを設定可能
